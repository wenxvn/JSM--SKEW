# 可交互新闻预览实施提示词

## 目标

为已完成基础布局的 SKEW 首页提供可见、可操作的新闻卡片预览。用户明确要求在 API 接入前看到接近参考图的完整效果，并要求导航、分类和新闻卡片均可点击，以便提前验证交互。

本次只构建本地界面预览，不读取或写入 Supabase，不抓取新闻，不调用 AI，不发送分析事件。所有预览内容必须在页面持续显示“界面预览”标识，不能与未来真实新闻混淆。未来 API 接入时，只替换预览数据输入，不重写筛选、详情查看、导航状态或卡片组件。

## 已阅读的技能

- `skills/develop/SKILL.md`：确认本次为 Next.js UI 与本地交互实现。页面组成、视觉方向和数据边界由用户需求、`AGENTS.md`、`design.md` 与本提示词明确。
- `skills/test/SKILL.md`：确认本次页面交互应优先以 lint、生产构建及浏览器手动操作验证。当前未配置测试框架，不为演示数据新增测试依赖。

## 已检查的代码与记录

- `AGENTS.md`：确认简体中文、Clerk 服务端认证、外部服务隔离和提示词批准流程。用户本次明确授权使用本地预览内容来查看页面效果。
- `design.md`：确认编辑式新闻视觉、颜色语义、响应式阈值、无障碍与默认真实数据优先原则。
- `app/page.tsx`：当前是保留 Clerk 认证的服务端页面，已实现顶部信息栏、品牌头部、静态分类行、内容区与页脚。
- `app/globals.css`：已有 `--skew-*` 令牌、三列网格、卡片基础类与响应式规则。
- `node_modules/next/dist/docs/01-app/03-api-reference/01-directives/use-client.md`：确认交互状态应放入独立的 `'use client'` 组件，服务端页面继续承担认证。
- `node_modules/next/dist/docs/01-app/03-api-reference/02-components/link.md`：确认真实页面跳转使用 `next/link`。本次不存在其他已实现页面，因此导航点击应切换当前页面内的预览状态，而不是链接到不存在的路由。
- 用户提供的参考图：参考其紧凑新闻卡片、图片比例、卡片层级和三列浏览节奏。不复制其文章、图片、英文内容、统计数据或品牌素材。

## 决策与假设

- 将交互逻辑拆到新的客户端组件，`app/page.tsx` 继续只做 Clerk 服务端认证和渲染入口。
- 新组件定义一个可替换的本地 `previewArticles` 数据数组。其字段对齐未来卡片输入：唯一标识、分类、地域、标题、摘要、预览视觉、来源计数、偏左中立偏右占比和简短详情。它不包含 URL、真实媒体名称、个人信息、真实统计或任何外部响应。
- 每张预览卡片和详情面板都明确带有“界面预览”标签。预览标题采用通用、非事件化的中文示例文案，不宣称为真实新闻。
- 卡片视觉不使用参考图、网络图片或第三方素材。使用本地 CSS 的受限中性色块与主题化抽象构图，确保页面可离线构建，并避免误导为真实新闻摄影。
- 顶部四个导航按钮均为原生 `button`。点击后切换当前导航状态，并以页面内说明反馈当前预览范围。分类项均为原生 `button`，点击后按分类过滤预览卡片；“全部”恢复完整列表。
- 点击新闻卡片或其“查看预览”按钮打开可关闭的原生语义对话框。对话框显示同一条预览数据的完整字段与视角标尺，关闭后返回原位置。点击遮罩和关闭按钮均可关闭，支持 Escape 键。
- 视角比例均为视觉预览值，不代表媒体立场或分析结论。每处都显示文字标签“预览偏左”“预览中立”“预览偏右”。
- 未来 API 接入时，保留组件的输入字段形状和交互状态，只用服务器端已保存数据替换 `previewArticles`。不在组件内添加抓取、AI、数据库或身份逻辑。
- 因用户明确要求预览数据，需同步修改 `design.md`，保留真实数据优先规则，同时新增一个严格限制的“界面预览例外”。该例外仅允许开发期、可见标识、不可伪装真实内容、不可默认在生产数据视图中使用的本地样例。

## 预计变更的文件

- `app/page.tsx`：保留服务端认证，改为渲染新的客户端新闻预览组件。
- `components/home-news-preview.tsx`：新增本地预览数据、分类与导航状态、卡片网格、详情对话框以及可访问键盘交互。
- `app/globals.css`：补充预览标识、可点击导航、卡片、视觉块、视角条、对话框和响应式样式，继续复用现有令牌。
- `design.md`：新增严格受限的开发期界面预览例外，并说明其显示与替换规则。

不修改数据库迁移、环境变量、依赖、Clerk 配置、Supabase、Oxylabs、AI、PostHog 或部署配置。

## 实施要求

### 页面与交互

- 保持完整简体中文界面。所有可见的示例内容必须用“界面预览”或同等清晰标识说明其性质。
- 在“头条新闻”标题旁显示简短预览提示，明确这些卡片用于验证布局和交互。
- 宽屏以三列网格展示至少九张预览卡片，约 900px 降为两列，约 640px 降为单列。卡片高度和预览视觉区域使用稳定尺寸，避免筛选后网格跳动。
- 卡片顺序为：抽象预览视觉、预览标签与分类、标题、摘要、三段视角条、来源数量与“查看预览”操作。不得使用真实人物肖像、真实事件、真实机构名称或真实新闻图像。
- 卡片整体可通过键盘聚焦并打开详情。卡片内部的“查看预览”按钮必须阻止重复触发，避免嵌套交互元素。
- 分类栏增加“全部”按钮。点击分类后高亮当前项并更新标题旁的筛选说明；没有结果时展示可恢复的中文空状态和“显示全部预览”按钮。
- 主导航按钮切换页面内预览状态。当前范围有明确视觉状态和屏幕阅读器语义。没有对应内容的范围可显示诚实的“预览准备中”状态，且提供回到“首页”的按钮。
- 详情对话框使用 `role="dialog"`、`aria-modal="true"`、可见标题及关闭按钮。打开时将焦点移入对话框，关闭时焦点回到触发卡片。支持 Escape、遮罩点击和关闭按钮。
- 不新增不存在页面的路由或伪造链接。品牌首页链接保持有效。

### 视觉与响应式

- 保持白色编辑式画布、细边框、4px 或 8px 圆角、轻量阴影和既有系统字体。禁止渐变、玻璃拟态、装饰性光球与营销式大卡片。
- 预览视觉区域不使用网络图片。采用通过 CSS 类决定的抽象主题画面，并在卡片内保留明确“界面预览”标识。
- 预览视角条只使用既有 `--skew-left`、`--skew-center`、`--skew-right` 令牌，且三个分段始终附带中文文本。
- 所有按钮至少具备 44px 高的触控区域或等效可用面积，具有 `:focus-visible` 焦点，并在 320px、390px 和宽屏中无重叠、裁切或全页横向滚动。
- 修改色彩、组件基调或响应式策略时同步 `design.md` 与 `app/globals.css`，不在 JSX 中散落颜色值。

## 安全要求

- 不读取、输出、写入或提交任何密钥、Cookie、令牌、环境变量值、用户资料、新闻源 URL 或外部响应。
- 不执行 Supabase 查询或写入、Clerk 管理、Oxylabs 抓取、AI 分析、PostHog 事件、Vercel 部署或生产环境操作。
- 本地预览数据不得被保存、上传、发出网络请求或用于授权判断。
- 预览数据必须永久有可见标识，不能被当作真实文章、真实来源、真实统计或真实分析结果展示。

## 验收标准

1. 首页在 API 未接入时可展示至少九张明确标为“界面预览”的中文新闻卡片。
2. 主导航、每个分类项、每张卡片、卡片详情操作、对话框关闭操作和无结果恢复操作都可以点击并具备键盘可访问性。
3. 分类点击后卡片数量与筛选说明正确变化，“全部”可恢复完整卡片列表。
4. 卡片详情可打开、可用 Escape、遮罩和关闭按钮关闭，关闭后焦点回到触发位置。
5. 页面保留 Clerk 服务端认证，未登录访问继续跳转到登录页。
6. 预览数据与真实数据边界清晰，无网络图片、真实新闻、真实来源、真实统计或外部调用。
7. 1440px、390px 与 320px 视口中的网格、对话框、标题、按钮和文字均无重叠、裁切或全页横向滚动。
8. `npm run lint`、`npm run build` 与 `git diff --check` 均通过。

## 需运行的检查

1. `npm run lint`
2. `npm run build`
3. `git diff --check`
4. 启动本地开发服务器，在 1440px、390px 和 320px 视口手动检查。
5. 使用鼠标和键盘分别操作主导航、分类、卡片、详情对话框、关闭按钮和无结果恢复操作。
6. 在浏览器网络面板确认上述交互没有 Oxylabs、AI、PostHog 或 Supabase 请求。

## 实施后的精确手动测试步骤

1. 登录开发环境账号并打开首页，确认“头条新闻”区域展示至少九张带“界面预览”标识的中文卡片。
2. 逐一点击顶部四个导航按钮，确认当前状态更新且任何未准备范围都有明确的页面内反馈。
3. 点击“国际”“科技”等分类项，确认卡片被过滤；点击“全部”确认九张预览恢复。
4. 使用键盘 Tab 聚焦一张卡片并按 Enter 或 Space，确认详情对话框打开；按 Escape 关闭并确认焦点回到原卡片。
5. 使用鼠标点击卡片的“查看预览”、对话框遮罩与关闭按钮，确认结果与键盘交互一致。
6. 在 1440px、390px 和 320px 检查三列、两列或单列网格，对话框、标签和按钮没有溢出。
7. 在网络面板确认不发生任何 API、数据库、抓取、AI 或分析事件请求。
8. 退出登录并访问首页，确认仍进入 Clerk 登录流程。

## 回滚路径

本次只涉及首页入口、客户端预览组件、全局样式与设计规则。若预览实现不再需要，移除客户端组件调用和新增样式，并恢复设计规范中的预览例外即可。没有数据库、认证或外部服务状态需要回滚。
