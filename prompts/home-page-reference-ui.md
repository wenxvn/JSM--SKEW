# 参考图首页界面实施提示词

## 目标

在既有受保护的 SKEW 首页上，依据用户提供的新闻首页参考图重构阅读界面。页面默认语言保持简体中文，视觉上吸收参考图的紧凑顶部信息栏、品牌导航、横向分类行、标题区和宽屏三列新闻阅读网格。

本次仅实现展示层。不得抓取新闻、调用 AI、读取 Supabase 或写入任何外部服务。由于尚未建立文章读取层，新闻网格必须以诚实的中文空状态替代虚构卡片、图片、新闻标题、来源数和分析结论。未来接入已保存文章时，页面结构应可直接承载真实卡片。

## 已阅读的技能

- `skills/develop/SKILL.md`：确认本次属于既有 Next.js 项目的 UI 改动。页面设计、数据来源和安全边界均已由 `AGENTS.md`、`design.md` 与本提示词明确，不引入新的架构决策。
- `skills/test/SKILL.md`：确认本次前端展示改动应以 lint、生产构建和多视口浏览器检查验证，不为静态样式新增无价值的测试。

## 已检查的代码与记录

- `AGENTS.md`：确认所有实施提示词必须使用中文，首页需保持 Clerk 服务端保护，界面只能展示已存储数据，且不触碰外部服务或生产环境。
- `design.md`：确认白色编辑式阅读基调、`--skew-*` 令牌、红蓝视角语义、中文排版、响应式阈值和真实数据优先原则。
- `app/page.tsx`：当前首页已通过服务端 `auth()` 保护，并提供中文品牌、静态分类外壳与空状态。
- `app/globals.css`：已定义运行时设计令牌、全局可访问性默认值、窄屏规则、文章卡片和偏向标尺基础样式。
- `app/layout.tsx`：根文档语言已为 `zh-CN`，并已包裹 `ClerkProvider`。
- `package.json`：现有 Next.js、React、Tailwind CSS 和 Clerk 依赖足以完成本次改动，不新增图标库或其他依赖。
- 用户提供的两张参考图：第一张明确要求首页包含顶部工具区、粘性站点头部、分类行、“头条新闻”与响应式三列新闻卡片。第二张提供了黑白编辑式视觉、信息密度和网格参考。两张图片仅作为视觉与布局方向，不复制英文文案、新闻、人物素材、图片或统计数据。

## 决策与假设

- 默认界面语言为简体中文，`SKEW` 保持品牌英文拼写，所有导航、状态和辅助文字使用中文。
- 保持现有 Clerk 服务端认证逻辑。未登录用户继续进入既有登录流程，已登录用户看到首页。
- 不添加参考图中的“浏览器扩展”“订阅”“登录”等未实现操作，也不添加指向不存在页面的导航链接。头部只呈现真实品牌、当前阅读位置和非交互状态信息。
- 分类项仅作为未来真实筛选接入前的静态信息提示，不承担筛选行为，并以“筛选功能接入后可用”等中文说明避免误导。
- “头条新闻”区域提供稳定的三列卡片网格容器和未来真实文章卡片的版式。无已保存文章时显示占满网格的空状态，不制造模拟文章、图片或来源数字。
- 视角标尺沿用现有“偏左，中立，偏右”三段式标签，红色和蓝色只承担分析语义，且始终由文字补充。
- 复用 `app/globals.css` 的 `--skew-*` 令牌。若为完成布局必须新增令牌、调整颜色、字阶、间距、圆角、阴影或响应式策略，必须同时修改 `design.md`；若只新增页面结构类且不改变系统规则，则无需修改长期规范。

## 预计变更的文件

- `app/page.tsx`：保留服务端认证，重组为中文的工具信息区、粘性品牌头部、分类提示行和“头条新闻”网格空状态。
- `app/globals.css`：补充对应布局、导航、分类行、网格和空状态样式，复用现有令牌及文章卡片基础类。
- `design.md`：仅在本次确实调整视觉令牌、组件基调或响应式策略时同步更新。

不修改数据库迁移、环境变量、依赖、Clerk 配置、Supabase、Oxylabs、AI、PostHog 或部署配置。

## 实施要求

### 视觉解读与布局

- 页面采用白色主画布和近黑正文，保留参考图中高密度而有秩序的编辑式阅读感。不要复制样图的图片、英文文本、品牌图形或页脚栏目。
- 顶部可增加一行极紧凑的只读信息条。其内容不得虚构准确日期、定位、国际版本状态或已安装浏览器扩展。使用不依赖外部数据的中文产品说明，例如“已保存新闻分析”。
- 品牌头部使用粘性定位、细底边框和白色背景。左侧放置 `SKEW` 与“新闻视角”，右侧使用非交互的“中文新闻分析”上下文，不添加误导性的按钮。
- 分类行置于品牌头部下方，使用可横向滚动的低对比小标签，保证窄屏下不换行挤压或产生全页横向滚动。标签必须清楚说明尚未可筛选。
- 主内容容器最大宽度为 1280px。区块标题使用“头条新闻”，不使用营销式首屏大标题。标题下方按宽屏三列、约 900px 两列、约 640px 单列组织内容区域。
- 空状态占用合理高度，使用细边框或极浅灰表面，说明尚未有“已保存且完成分析”的文章，以及真实数据接入后会显示的内容。它不能承诺抓取何时完成，也不能将尚未实现的操作包装成按钮。
- 预留的真实文章卡片应按来源与时间、标题、摘要、框架标签、三段式视角条和必要操作排列。卡片只在真实数据可读时渲染，不在本次使用硬编码内容。
- 页面可保留克制页脚，仅含品牌、产品说明和版权年份。若年份不是动态或受控的真实值，应避免精确年份；不要展示虚构公司、帮助、社交媒体或法律链接。

### 中文排版、颜色与响应式

- 使用既有系统中文无衬线字体栈，品牌可保持更高字重。不得使用负字距、远程字体、渐变、玻璃拟态、装饰性光球或大圆角。
- 文字层级遵循 `design.md`，区块标题约 20 至 24px，卡片标题约 18 至 20px，正文 15 至 16px，导航与元信息 12 至 13px。
- 使用既有四像素间距令牌、1px 边框、4px 或 8px 圆角和轻量阴影。不得在 JSX 或局部样式中散落等价色值。
- 在 1440px 宽屏时，主网格稳定为三列。在 900px 以下切换为两列或一列，在 640px 以下为单列。320px、390px 与宽屏下均不得文字重叠、裁切、全页横向滚动或不合理留白。
- 所有实际链接和按钮保留可见键盘焦点。静态分类项使用非交互语义，不伪装成可点击控件。

## 安全要求

- 不读取、输出、写入或提交任何密钥、Cookie、令牌、环境变量值、用户资料或新闻源 URL。
- 不创建模拟新闻、来源、统计、图片、分析结论或人物信息。页面只可渲染本地界面结构与当前真实的无数据状态。
- 不执行 Supabase 查询或写入、Clerk 管理操作、Oxylabs 抓取、AI 分析、PostHog 事件发送、Vercel 部署或生产环境操作。
- 保持认证和授权判断在 `app/page.tsx` 服务端边界，不能将其下放到客户端展示代码。

## 验收标准

1. 根文档与首页可见文本均以简体中文呈现，`SKEW` 保留品牌拼写。
2. 已登录首页具有参考图启发的紧凑信息条、粘性品牌头部、分类行和“头条新闻”网格结构，但不复制其文章、图片、英文或功能。
3. 未接入文章读取层时，网格只显示诚实空状态，不显示虚构新闻卡片、来源数、偏向比例或占位人物图片。
4. 布局使用现有设计令牌，呈现白色编辑式阅读界面、细边框、克制圆角和只用于视角语义的红蓝色。
5. 未登录访问继续遵循既有 Clerk 登录重定向，且本次不改变认证配置。
6. 在 1440px、390px 和 320px 视口中，页面无重叠、裁切和全页横向滚动，分类行可单独水平浏览。
7. 不产生数据库、外部服务、依赖或生产配置变更。

## 需运行的检查

1. `npm run lint`
2. `npm run build`
3. `git diff --check`
4. 使用本地开发服务器分别检查宽屏 1440px、390px 与 320px 页面。
5. 检查未登录跳转和已登录首页，确认静态标签没有表现为可用筛选。

## 实施后的精确手动测试步骤

1. 启动开发服务器，使用开发环境测试账号登录并访问 `http://localhost:3000`。
2. 确认顶部、品牌、分类说明、“头条新闻”和空状态均为简体中文，且没有英文新闻文案或参考图中的人物图片。
3. 在 1440px 宽度检查粘性头部、内容容器、区块标题与三列网格空状态的比例和细边框层级。
4. 在 390px 与 320px 宽度检查头部文字换行、分类行独立横向滚动、主内容单列、空状态文字和焦点显示，确认没有全页横向滚动、重叠或裁切。
5. 未登录后重新访问首页，确认仍进入 Clerk 登录流程。
6. 打开浏览器网络面板，确认首页不发起 Oxylabs、AI、PostHog 或 Supabase 写入请求。

本次只涉及首页展示文件、全局样式和可能同步的设计规范。若出现视觉或构建回归，恢复这些受版本控制文件到实施前状态即可。不存在数据库、认证或外部服务状态需要回滚。
