# 新闻详情页实施提示词

## 目标

为 SKEW 新增受保护的 `app/news/[id]/page.tsx` 新闻详情页。首页每张“界面预览”新闻卡片点击后必须进入其对应的详情地址，而不再打开页面内对话框。

详情页以用户提供的图 1 为信息密度和阅读结构参考，形成共享的顶部信息栏与站点头部、文章正文主栏、分析侧栏、相关新闻网格、订阅提示带和共享页脚。首页和详情页都使用本地的、带明确“界面预览”标识的类型化样例数据和合成新闻封面，以便在 Supabase 数据层完成前验证真实阅读产品的视觉效果。不得抓取、分析、写入数据或调用业务外部服务。

本次补充改善首页新闻卡片的视觉完成度。用户提供的 `image copy.png` 和 `image copy 2.png` 只作为结构与密度参考，不直接用于网站。首页不再以纯文字和 CSS 色块作为主要封面，而是为每条本地预览文章使用一张从允许网站使用的图片库下载并保存到项目内的通用编辑图片。

## 决策与假设

- 新路由使用 `/news/<preview-id>`。`preview-id` 直接来自现有预览文章的稳定 `id` 字段，首页卡片 `Link` 与详情页解析使用同一份数据，不根据标题或位置推导地址。
- 将 `ArticlePreview` 类型和 `previewArticles` 移至一个无副作用、可被服务端与客户端共同导入的本地模块，例如 `lib/preview-articles.ts`。新增的查找函数只解析本地数组，不读取数据库、环境变量或网络。
- 详情页服务端先执行 Clerk `auth()`，未认证时沿用首页的登录重定向。认证后解析 `params.id`；无对应预览文章时使用 Next.js 的 `notFound()`，不展示错误的默认文章。
- 首页保留分类和导航预览交互，但移除 `selectedArticle`、焦点陷阱和对话框。每张卡片整体使用一个有效的 `Link` 跳转到对应详情页，避免嵌套的链接与按钮。
- 详情页及其所有示例字段持续显示“界面预览”标识和说明。标题、摘要和正文使用简短、自然的中文合成新闻文案，不使用真实新闻、来源、人物、日期、数据或分析结论。页面只能显示本地样例，不能暗示为已保存文章。
- 为全部预览文章搜集一组允许网站使用的本地编辑封面图片，使用 `public/preview-news/` 下的稳定文件名。图片内容覆盖城市公共空间、葡萄园、科研仪器、社区图书馆、抽象全球协作、健康服务、运动场、城市气候设施和文化空间等通用主题。每张图均为 16:9 或可安全裁切为 16:9 的横向构图，无文字、无水印、无商标、无国旗、无政治人物、无新闻机构标识、无可识别面孔，也不复刻用户提供的参考图。
- 图片下载前后都必须使用“界面预览”数据边界。素材不是当前预览文章所描述的真实新闻现场，卡片和详情页主图都要以文字说明其为“预览封面图”。
- 将 `design.md` 的界面预览例外同步调整为允许上述严格受限的、已下载到本地的通用编辑图片。规则必须继续禁止真实新闻图像、真实人物、真实事件、真实来源和未标识的演示数据。
- 本次不实现收藏、分享、订阅提交、反馈、查看全部来源或其他未接通行为。只保留可工作的“返回首页”和相关新闻链接。订阅带是明确说明“订阅功能尚未接入”的展示区，不渲染会误导用户的可提交表单。
- “视角分析预览”“摘要预览”“来源构成预览”使用同一文章的本地字段派生显示。红蓝色只用于带“预览偏左”“预览中立”“预览偏右”文字的语义标尺，不是事实判断。
- 所有页面共用的顶栏、站点头部与页脚应抽为可复用展示组件，确保首页与详情页使用相同的结构和视觉规则。组件不能包含认证、数据库、抓取或分析逻辑。
- 本次新增的是遵循既有令牌的页面样式，不调整任何全局颜色、字体、间距、圆角、阴影或响应式规则。若实施中确实必须改动这些长期规则，必须同步修改 `design.md` 与 `app/globals.css`。

## 实施要求

### 路由、认证与数据边界

1. 在 `app/news/[id]/page.tsx` 创建动态路由。`params` 必须按当前 Next.js 约定以 `Promise<{ id: string }>` 读取。
2. 详情路由必须与首页相同先进行 Clerk 服务端认证。不要把认证状态传入客户端来代替服务端检查。
3. 用共享本地数据模块解析文章。文章不存在时调用 `notFound()`，不能把任意无效 ID 映射到第一篇预览文章。
4. 所有预览文章内容、视角百分比、来源数、摘要要点和相关推荐只来自本地类型化数组或该数组的确定性派生，且不发生数据库读写或外部请求。
5. 为未来 Supabase 接入保留清晰的输入边界。届时只替换共享数据模块为服务端已保存数据映射，不重写卡片、详情布局或分析面板组件。

### 预览素材与首页卡片视觉

1. 从允许网站使用的图片库搜集每个预览文章所需的一张通用编辑封面。下载前检查来源许可和图片内容，图片不得含有文字、水印、商标、可识别人物、政治人物、旗帜、新闻机构标志或可归属的真实新闻事件。
2. 下载后逐张检查主体、构图和裁切。只把通过检查的文件保存到 `public/preview-news/`，并在最终回复中列出保存路径、图片主题和来源平台。不得使用用户提供的截图作为网站内容，也不得在运行时直接引用第三方图片地址。
3. 扩展共享文章类型，为每条文章提供本地 `imageSrc` 和可读的 `imageAlt`。`imageAlt` 必须以“预览封面图”开头，并描述通用场景，不将其称作真实新闻现场。
4. 首页卡片封面使用 `next/image` 或等效优化图片组件。桌面封面固定为 16:9，使用 `object-fit: cover`，图片加载不改变卡片尺寸。卡片上叠放低对比但清晰可见的“界面预览”标识，不能使用遮挡主要画面的厚重蒙版。
5. 卡片内容采用紧凑的编辑式节奏，依次为分类与地域、最多两行的自然中文标题、简短摘要、固定高度的三段视角条和来源数量。预览标题和摘要本身不得再出现“用于展示”“验证布局”等开发说明，这些说明只放在页首提示与预览标识中。
6. 封面、标题和信息条组成单一可点击 `Link`。悬停时只使用细微边框和阴影变化，不能缩放图片或改变网格尺寸。
7. 更新 `design.md` 和 `app/globals.css` 中的界面预览素材规则，使用统一的 `--skew-preview-*` 令牌管理预览标识与可选叠层，不能在 JSX 内散落颜色值。设计说明必须明确图片只用于预览版的视觉效果，不是相应文章的真实报道照片。

### 首页跳转

1. 保留现有首页分类与导航预览状态。
2. 将每张预览卡片从会打开对话框的 `button` 改为 `Link`，目标为 `/news/${article.id}`。链接具备清晰中文无障碍名称。
3. 删除只服务于对话框的客户端状态、事件监听和对话框 DOM，避免保留无法到达的代码。
4. 卡片内“查看预览”提示改为链接内的静态视觉提示。整个卡片是一个链接，内部不得嵌套交互元素。

### 页面组成与视觉

1. 顶部使用共享的深色细信息栏，正文页站点头部使用共享品牌、导航和当前“新闻详情”上下文。品牌链接始终回到首页。
2. 内容区最大宽度保持 1280px，宽屏采用约 `minmax(0, 1.7fr)` 主栏与 `minmax(280px, 0.85fr)` 侧栏。两栏间用稳定的 24px 至 32px 间距，避免卡片套卡片。
3. 主栏依次展示：预览标签与分类地域，文章标题，简短的预览元信息，合成主视觉与其预览说明，三段式视角分布条，以及多个语义化正文段落。正文内容必须持续表明为展示样例，不能陈述未验证的真实事实。
4. 侧栏按阅读顺序展示三个独立面板。
   - “视角分析预览”显示总体视角、三条有文字标识的比例条和简短的方法说明。
   - “摘要预览”显示明确标识的概览条目和“仅用于界面验证”的说明。
   - “来源构成预览”显示抽象的来源类别及视角文字，不使用真实媒体名称。
5. 正文末尾显示“相关新闻预览”，排除当前文章并显示不超过四篇其他本地预览文章。每张项目为实际 `Link`，可跳转到对应详情页。
6. 相关新闻下方展示无框订阅提示带，保留标题与“订阅功能尚未接入”的明确状态文本。不得放置能收集邮箱或提交请求的控件。
7. 详情页底部使用共享页脚，并继续标明是界面预览，未包含真实新闻数据。
8. 仅使用白色画布、细边框、4px 或 8px 圆角、轻量阴影、系统字体和现有颜色令牌。禁止渐变、玻璃拟态、大圆角、无语义红蓝背景、装饰性光球或营销式布局。

### 响应式与无障碍

1. 约 900px 以下将详情页从两栏改为单列，侧栏顺序紧跟正文主栏。相关新闻转为两列或一列，文字不能溢出。
2. 约 640px 以下维持稳定的水平内边距，标题自然换行，元信息和视角条在需要时按行排列，相关故事为单列。
3. 在 320px、390px 和宽屏下不得出现横向滚动、内容覆盖、文字裁切或被隐藏的关键标签。
4. 所有链接和按钮具备可见 `:focus-visible` 状态与可读名称。视角颜色以明确文字标签补足，图片型预览视觉标记为装饰性或提供合适替代文字。
5. 标题、主体、侧栏和相关内容使用语义化 `main`、`article`、`aside`、`section`、`nav`、`h1` 至 `h3` 等结构，保证屏幕阅读器的阅读顺序。

## 安全要求

- 不读取、输出、写入或提交密钥、Cookie、访问令牌、环境变量值、个人资料、新闻源 URL 或外部响应。
- 不执行 Supabase 查询或写入、Clerk 管理操作、Oxylabs 抓取、AI 分析、PostHog 自定义事件、Vercel 部署或任何生产环境操作。
- 本地预览内容不能保存、上传、发送到网络或用于认证和授权判断。
- 不使用参考图片中的新闻图片、真实文章、人物、日期、来源或统计数据。预览标识在首页、详情、侧栏、相关内容与页脚都必须可见。
- 下载的项目素材必须来自允许网站使用的图片库，并且不得含有可读文字、水印、商标、真实人物肖像、政治人物、旗帜、可识别新闻事件、新闻机构标志或受版权保护的角色。每一张素材只能作为通用预览封面，不能暗示为对应文章的真实报道照片。

## 验收标准

1. 已认证用户点击首页任意新闻预览卡片，会进入 `/news/<preview-id>` 对应详情页，浏览器前进和后退行为正常。
2. 未认证用户访问首页或任意 `/news/<preview-id>` 地址，继续进入 Clerk 登录流程。
3. 有效详情 ID 显示与首页对应的标题、分类、抽象视觉和预览分析。无效 ID 显示 Next.js 404，而不显示错误文章。
4. 详情页包含共享顶栏和站点头部、两栏文章与侧栏、相关新闻、订阅提示带和共享页脚。
5. 首页卡片和详情页主图各使用对应的本地 16:9 预览封面图，图片比例、裁切、加载和响应式行为稳定，不出现纯文字卡片、破图、文字水印或意外变形。
6. 每个内容块清晰标记为“界面预览”或等效文本，页面没有真实新闻、真实来源、人物图片、真实统计、数据库读取或外部调用。
7. 所有相关新闻链接可跳到正确详情页，品牌链接可返回首页。
8. 1440px、390px 和 320px 下的文章、侧栏、标题、视角条、相关内容和页脚均无重叠、裁切或全页横向滚动。
9. `npm run lint`、`npm run build` 和 `git diff --check` 均通过。

## 需运行的检查

1. `npm run lint`
2. `npm run build`
3. `git diff --check`
4. 启动 `npm run dev`，在登录后的开发环境中检查首页卡片跳转、直接详情地址、无效 ID 与相关新闻跳转。
5. 在 1440px、390px 和 320px 宽度检查首页及详情页的布局、文字、焦点和无横向滚动。
6. 使用浏览器网络面板确认本次路由与展示不新增 Supabase、Oxylabs、AI 或 PostHog 自定义请求。
7. 检查每张下载封面的许可、可读性、比例、裁切、无文字水印和无真实新闻标识，并确认它们都从项目内 `public/preview-news/` 加载。

## 实施后的精确手动测试步骤
