# SKEW 界面设计规范

## 定位与适用范围

SKEW 是面向简体中文读者的新闻分析产品。界面采用克制、清晰的编辑式阅读风格，以阅读、比较和理解已保存的新闻分析为中心。

本规范适用于页面、组件、全局样式、空状态和界面相关实施提示词。它提炼自已批准的视觉方向，不复制任何参考图中的内容、新闻、素材或页面布局。

`app/globals.css` 是本规范令牌的运行时实现，`design.md` 是长期规则来源。涉及视觉令牌、组件基调或响应式策略的改动，必须在同一变更中更新二者。

## 设计原则

- 信息优先。阅读层级、文章来源、分析结果和操作状态应比装饰更醒目。
- 真实数据优先。界面只展示已保存的数据，数据未接通时使用明确空状态，不以虚构文章、来源、统计数、人物图像或分析结论填充页面。
- 克制装饰。使用白色画布、细边界、大留白和轻量阴影建立层级。
- 立场不是事实判断。红蓝色只辅助解读分析中的视角位置，不代表事实结论或媒体定性。
- 先保证可访问性与响应式。文字、焦点、语义标签和小屏排版是组件完成的必要条件。

## 颜色与语义

所有界面色彩优先引用 `app/globals.css` 中的 `--skew-*` 变量。不要在 JSX、局部样式或单个组件中重复散落等价色值。

| 变量 | 当前值 | 用途 |
| --- | --- | --- |
| `--skew-canvas` | `#FFFFFF` | 主画布与主要卡片背景 |
| `--skew-surface` | `#F7F7F7` | 次级背景与空状态 |
| `--skew-surface-strong` | `#F0F1F3` | 更强调的中性表面 |
| `--skew-text` | `#0D0D0F` | 主标题、正文和重点信息 |
| `--skew-text-muted` | `#6E7280` | 元信息、说明和次级内容 |
| `--skew-border` | `#E5E7EB` | 卡片和控件边框 |
| `--skew-divider` | `#ECEEF0` | 模块内部分隔线 |
| `--skew-left` | `#B42318` | 偏左视角的语义色 |
| `--skew-center` | `#E5E7EB` | 中立视角的语义色 |
| `--skew-right` | `#1D4ED8` | 偏右视角的语义色 |
| `--skew-focus` | `#0D0D0F` | 键盘焦点轮廓 |

红色和蓝色不得作为无语义的强调、链接颜色或装饰背景。视角信息必须同时显示文字标签，例如“偏左”“中立”“偏右”，不能只依赖颜色传达。

## 排版

- 根文档语言使用 `zh-CN`。
- 正文使用 `--skew-font-ui` 系统无衬线字体栈。它覆盖 `PingFang SC`、`Hiragino Sans GB`、`Microsoft YaHei`、`Noto Sans CJK SC` 等常见中文字形，并为英文、数字和品牌提供通用回退。
- 不引入需要浏览器或构建时下载的远程字体。字体加载必须保持离线构建稳定。
- 不使用负字距。中文标签、导航和正文保持正常字距。

| 层级 | 推荐字号 | 字重 | 行高 |
| --- | --- | --- | --- |
| 品牌 | 25px | 700 至 750 | 1.0 |
| 页面主标题 | 30 至 40px | 700 至 750 | 1.25 |
| 区块标题 | 20 至 24px | 700 至 750 | 1.35 |
| 卡片标题 | 18 至 20px | 700 至 750 | 1.35 |
| 正文 | 15 至 16px | 400 | 1.6 至 1.65 |
| 元信息与标签 | 12 至 13px | 500 至 700 | 1.4 至 1.5 |

紧凑面板中使用区块标题或更小层级，不在卡片、侧栏或工具区域使用营销式大标题。

## 间距、边界与层级

- 使用 `--skew-space-*` 四像素倍数令牌。常用值为 4、8、12、16、20、24、32、40 和 48px。
- 内容容器最大宽度为 1280px。桌面与移动端均要保留稳定的水平内边距。
- 使用 `--skew-border` 的 1px 细边框和 `--skew-divider` 的内部区隔建立秩序。
- 圆角只使用 `--skew-radius-small` 的 4px 或 `--skew-radius-medium` 的 8px。不要使用大圆角或胶囊式大容器。
- 阴影只使用低对比的 `--skew-shadow-small` 或 `--skew-shadow-medium`，不能以浓重阴影制造主要层级。
- 不使用渐变、玻璃拟态、装饰性光球、散景背景或营销式巨型卡片。

## 布局与响应式

- 宽屏以居中容器和稳定网格组织内容。固定格式工具、偏向条和标签需要稳定尺寸，不能因内容或悬停状态改变布局。
- 约 900px 以下减少网格列数，约 640px 以下改为单列。
- 最小支持宽度为 320px。任何视口中均不得产生横向滚动、文字裁切、重叠或无意义的空白。
- 正文与标题字号不随视口宽度无限缩放。仅允许在明确的设计范围内使用受限的 `clamp()`，例如当前页面主标题的 30 至 40px 范围。
- 文字过长时优先换行或调整布局，不用省略号隐藏页面标题、状态或关键解释。

## 组件规则

### 卡片与面板

- 卡片用于独立新闻、模态框和确实需要框定的工具。页面区段本身保持无框或使用整页分隔，不把卡片嵌套在卡片中。
- 新闻卡片应依次呈现来源与时间、标题、摘要、框架标签、视角信息和必要操作。未来接入数据时必须使用已保存字段。
- 面板标题与内容以细分隔线分开，标题层级要小于页面主标题。

### 分类、按钮与链接

- 分类与框架标签使用小型、低对比的 4px 圆角标签，不应伪装成已可用的筛选控件。
- 只能用原生按钮或链接表达实际操作。未实现的筛选与功能使用静态说明或禁用状态，并明确说明其当前状态。
- 可交互元素必须具有可见 `:focus-visible` 焦点、足够的点击面积和文字说明。图标按钮需要无障碍名称。

### 视角标尺与空状态

- 视角标尺采用偏左、中立、偏右三段式语义。每段配合文字标签和必要说明。
- 空状态说明没有什么数据、为什么没有，以及后续何时会出现，不承诺未实现功能。
- 空状态不能以演示文章或伪造统计数字替代真实结果。

## 内容与数据边界

- 界面只能读取并展示已存储的数据，不能直接抓取新闻、执行 AI 分析或修改管道状态。
- 用户身份、授权、密钥和数据库写入始终留在服务端边界，不能由视觉层承担。
- 不在界面、样式、设计文档或提示词中记录密钥、Cookie、令牌、新闻源 URL 或用户直接标识信息。

## 实现与验证

每次界面改动应执行以下检查：

1. 先阅读本文件与 `app/globals.css`，复用 `--skew-*` 令牌和既有组件模式。
2. 运行 `npm run lint` 与 `npm run build`。
3. 在宽屏、390px 和 320px 视口检查文字、网格、空状态、焦点和视角标签。
4. 确认颜色对比足够，红蓝语义有文字补充，未实现控件不会误导。
5. 确认没有新增外部调用、虚构数据或客户端敏感信息。

## 变更规则

- 影响颜色、字体、间距、圆角、阴影、组件基调或响应式策略的改动，必须同步更新本文件与 `app/globals.css`。
- 影响页面组成或交互状态的改动，应在对应实施提示词中写明视觉与手动验收要求。
- 引入深色模式、第二套品牌主题或新的语义色前，必须先单独定义对比度、状态映射和迁移策略，不能临时覆盖现有令牌。
